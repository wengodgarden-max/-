## 二、各渠道职责

| 渠道 | 角色 | 职责 |
|------|------|------|
| **小红书** | 流量入口 | 发笔记、引流、建立信任 |
| **微信** | 成交 + 私域 | 客服、收款、发激活码、售后 |
| **飞书** | 管理 + 自动化 | 订单管理、用户管理、客服机器人、数据统计 |
| **网页** | 产品交付 | 用户实际使用AI对话 |

## 四、飞书具体功能

### 4.1 飞书多维表格（数据管理）

**创建以下表格：**

#### 表1：订单表
```
字段：
- 订单ID（自动编号）
- 用户微信名（文本）
- 用户微信号（文本）
- 套餐类型（单选：体验版/标准版/年度会员）
- 金额（数字）
- 激活码（文本）
- 状态（单选：未付款/已付款/已使用/已过期）
- 创建时间（创建时间）
- 备注（多行文本）
```

#### 表2：激活码表
```
字段：
- 激活码（文本）
- 套餐类型（单选）
- 使用次数限制（数字）
- 已使用次数（数字）
- 状态（单选：未分配/已分配/已使用/已过期）
- 分配给用户（文本）
- 分配时间（日期）
- 过期时间（日期）
```

#### 表3：用户反馈表
```
字段：
- 用户名（文本）
- 反馈内容（多行文本）
- 满意度（单选：非常满意/满意/一般/不满意）
- 处理状态（单选：待处理/处理中/已解决）
- 处理结果（多行文本）
```

### 4.2 飞书机器人（自动通知）

**场景1：新订单通知**
```
当微信客服手动在多维表格添加订单时
→ 飞书机器人自动发消息到群

消息内容：
🔮 新订单提醒
用户：张三
套餐：体验版
金额：¥9.9
请及时发送激活码！
```

**场景2：使用提醒**
```
当用户激活码快过期时（如7天后）
→ 飞书机器人发提醒

消息内容：
⏰ 激活码即将过期提醒
用户：张三
激活码：ALCH-TRIAL-A001
剩余天数：7天
请提醒用户及时使用！
```

### 4.3 飞书 + 微信联动

**方案A：手动同步**
```
微信成交 → 手动在飞书多维表格记录订单 → 飞书机器人通知
```

**方案B：自动化同步（推荐）**
```
微信成交 → 调用飞书API自动创建订单 → 飞书机器人通知
```
